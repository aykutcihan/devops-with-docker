# Exercise 3.2 - A Deployment Pipeline to a Cloud Service

Extend Exercise 1.16 (Render.com deployment) with an automated pipeline. Every push to GitHub should trigger a new deployment to the cloud service.

## Requirements

- Add a GitHub Actions workflow that deploys to your cloud service on every push
- Look for existing GitHub Actions for your cloud provider (e.g. Render)
- The repository README must include a link to the live deployed application

Submit a link to the repository. README must contain the deployed app URL.
